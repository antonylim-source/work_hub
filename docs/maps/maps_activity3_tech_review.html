01 프로젝트 개요

POLY M.A.P.S. 4개 Activity 중 개발 우선순위 1위. 외부 API 불필요, 완전 온디바이스 동작으로 4개 중 기술 난이도가 가장 낮아 즉시 착수 가능. 합성어(compound word) 58개를 기반으로 학습자가 터치/마우스 슬래시로 형태소 경계를 절단하는 게임형 학습 콘텐츠.

뇌과학 근거: 물리적 절단 체험으로 긴 단어에 대한 인지 공포 해소. 시각 피질에 형태소 단위 자동 분석 회로(0.1초 처리)를 이식하는 교육 효과.

03 확정 기술 스택
스택 확정 경위: 초기 Vanilla JS + Vite 6.x + Pointer Events API로 설계 → Cursor 검토 후 TypeScript + React 19 + Vite 8.x + Touch/Mouse Events API로 전환 확정.

확정 스택 (즉시 적용)

TypeScript React 19 Vite 8.x Touch Events API Mouse Events API GitHub Pages (배포)

이펙트 강화 검토 옵션 (Phase 2)

Phaser.js (게임 이펙트) 파티클 이미터 물리 엔진 (조각 튕김)

Phaser는 레이저 히트 이펙트, 형태소 절단 애니메이션, 화면 쉐이크 등 비주얼 강화에 유리. 현재 React 스택과 병행 검토 중.

04 난이도 시스템

저연령 아동의 소근육 발달 수준을 고려한 3단계 허용 오차 설계. 어드민 설정에서 캠퍼스별 조정 가능.

난이도 허용 오차 대상 연령 비고
Easy ±20px 저연령 (유아~초등 저학년) 소근육 발달 단계 배려
Normal ±18px 초등 중학년 기본 난이도
Hard ±12px 초등 고학년 이상 정밀 조작 요구
어드민 설정 가능 항목: 허용 오차(px), 세션당 단어 수, Miss 한도, 콤보 보너스 배율. 캠퍼스·학년별 맞춤 설정 지원 예정.
05 문제점 & 해결 방안
🔴 문제 ① — 페널티 과중 및 저연령 소근육 조작 난이도
오타 시 괴물 1.5× 거대화 페널티는 유아에게 불안감·학습 거부 반응 유발 가능. 저연령의 소근육 발달 미완성으로 인한 과도한 오타 누적 우려.
✅ 해결 방안
단계적 힌트 시스템 도입: 1차 오타 → 경계 영역 하이라이트 표시 / 2차 오타 → 절단선 가이드 제공 / 3차 오타 → 자동 정답 처리 후 다음 단어 진행.
Easy 모드 전용 설정: 페널티 비활성화 옵션 + ±20px 관대한 판정 적용.
구현: Cursor 가이드 STEP 5 — 힌트 3단계·난이도 선택
🔴 문제 ② — 이펙트/비주얼 완성도 (현재 POC 수준)
현재 POC는 기능 검증 수준. 레이저 빔 드로잉, 파티클 히트 이펙트, 형태소 조각 튕김 등 상용 수준의 비주얼이 미구현 상태.
✅ 해결 방안
Phase 1 (현 스택 내): CSS/Canvas 기반 레이저 궤적 드로잉 + 기본 파티클 효과 구현.
Phase 2 검토: Phaser.js 도입 — 파티클 이미터, 물리 엔진 기반 조각 튕김, 글로우 필터, 화면 쉐이크. 현재 React 스택과 레이어 분리 방식으로 통합 가능.
