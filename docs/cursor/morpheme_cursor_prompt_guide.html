1
타입 기반 확립 — SlashEvent & BoundaryResult
판정 로직의 입출력 계약 먼저 정의. 이후 모든 프롬프트가 이 타입에 의존.
타입 정의
왜 1번인가: 터치 이벤트는 TouchEvent와 MouseEvent 두 갈래인데, 판정 로직은 둘 다 동일한 입력 형식을 받아야 함. 공통 타입을 먼저 만들면 Cursor가 이후 단계에서 타입 불일치로 헤매지 않음.
Cursor에 붙여넣을 프롬프트
// Context: POLY M.A.P.S. Morpheme Laser Cutter // Stack: TypeScript + React 19 + Vite 8.x // File to modify: src/types/game.types.ts Add the following types to game.types.ts for the touch/slash detection system: 1. SlashGesture — normalized input from BOTH TouchEvent and MouseEvent: - startX: number // gesture start X (clientX) - startY: number // gesture start Y (clientY) - endX: number // gesture end X - endY: number // gesture end Y - durationMs: number // total gesture duration in ms - isTouch: boolean // true = TouchEvent, false = MouseEvent 2. BoundaryHit — result of one slash judgment: - wordId: string // e.g. "raincoat" - boundaryIndex: number // which boundary was targeted (0-based) - boundaryPixelX: number // expected X position on screen - slashX: number // actual slash X position - delta: number // Math.abs(slashX - boundaryPixelX) - tolerance: number // allowed tolerance for current difficulty - isHit: boolean // delta <= tolerance - difficulty: 'easy' | 'normal' | 'hard' 3. Difficulty config object: - easy: tolerance 20px (ages 4-6, low motor control) - normal: tolerance 18px (ages 7-9) - hard: tolerance 12px (ages 10+) Do NOT create any components or hooks yet. Types only.
핵심 포인트: SlashGesture가 TouchEvent/MouseEvent를 추상화하는 게 핵심. 이후 판정 훅은 이벤트 타입을 몰라도 됨. delta를 타입에 포함시키면 디버그 오버레이 구현이 쉬워짐.
기대 출력물
  • game.types.ts에 3개 타입 추가, 기존 타입 유지
  • DIFFICULTY_CONFIG 상수 객체 (tolerance 값 3종)
2
입력 정규화 — useSlashInput 훅
Touch/Mouse 이벤트를 동일한 SlashGesture로 변환. 판정 로직과 분리.
입력 추상화
왜 분리하나: 판정 로직(useSlashJudge)이 이벤트 타입을 알 필요 없음. 나중에 Phaser로 교체하거나 게임패드를 추가할 때 이 훅만 교체하면 됨.
⚠ 모바일에서 touchstart → touchend 사이에 스크롤이 발생하면 판정이 오염됨. preventDefault()와 passive 이벤트 처리를 명시적으로 요청할 것.

// Context: POLY M.A.P.S. Morpheme Laser Cutter // Types already defined in game.types.ts (SlashGesture, BoundaryHit, DIFFICULTY_CONFIG) // Create: src/hooks/useSlashInput.ts Create a React hook useSlashInput(ref: RefObject) that: INPUT HANDLING: - Attaches BOTH touch and mouse listeners to the given ref element - Touch: touchstart → record start, touchend → compute gesture - Mouse: mousedown → record start, mouseup → compute gesture - On touchstart: call event.preventDefault() to block scroll interference (attach with { passive: false }) - Ignore multi-touch (touches.length > 1) → return null NORMALIZATION: - Extract clientX/clientY from both event types identically - Compute durationMs = endTimestamp - startTimestamp - Set isTouch = true for touch events, false for mouse - Return a SlashGesture object (from game.types.ts) FILTERING (reject invalid gestures): - Minimum drag distance: 30px horizontal movement required - Maximum duration: 800ms (longer = accidental drag, not slash) - Vertical tolerance: Math.abs(deltaY) < Math.abs(deltaX) * 1.5 (reject near-vertical swipes) RETURN VALUE: - lastGesture: SlashGesture | null - isSlashing: boolean (true during active gesture) Export as default. No UI. Pure input abstraction only. Cleanup all event listeners on unmount.
필터링 기준 근거: 30px 최소 거리는 아동이 화면 터치 시 발생하는 자연적인 손떨림 제거용. 800ms 제한은 "천천히 드래그"와 "빠른 슬래시"를 구분. 수직 방향 필터는 스크롤 제스처 오인식 방지.
기대 출력물
  • useSlashInput.ts 훅 — SlashGesture 반환
  • Touch/Mouse 이벤트 리스너 자동 정리 (cleanup)
  • 멀티터치 무시 처리
3
핵심 판정 엔진 — useSlashJudge 훅
|slashX − boundaryPixelX| ≤ tolerance 공식 구현. 난이도별 분기.
핵심 로직
판정 공식: isHit = Math.abs(gesture.endX - boundaryPixelX) <= DIFFICULTY_CONFIG[difficulty].tolerance
단, boundaryPixelX는 화면 렌더링 좌표이므로 DOM getBoundingClientRect()로 실시간 계산해야 함 (폰트 크기나 반응형 레이아웃 변화 반영).
⚠ 흔한 실수: boundaryPixelX를 하드코딩하면 화면 크기 변경 시 판정이 어긋남. 반드시 단어 DOM 요소의 getBoundingClientRect()로 동적 계산 요청할 것.

// Context: POLY M.A.P.S. Morpheme Laser Cutter // useSlashInput hook is complete (returns SlashGesture) // Create: src/hooks/useSlashJudge.ts Create useSlashJudge hook that takes a SlashGesture and judges boundary hits: SIGNATURE: useSlashJudge( wordRef: RefObject, // the word display element wordData: CompoundWord, // current word with boundary data difficulty: 'easy' | 'normal' | 'hard', gesture: SlashGesture | null ): BoundaryHit | null BOUNDARY PIXEL CALCULATION: - wordData has morphemes: string[] e.g. ["rain", "coat"] - Render the word in the DOM, then use getBoundingClientRect() on each character span to find where each morpheme boundary falls - boundaryPixelX = right edge of morpheme[0] span (= left edge of morpheme[1]) - This must be calculated AFTER the word renders, not hardcoded JUDGMENT: - slashX = gesture.endX (final X position of the slash) - delta = Math.abs(slashX - boundaryPixelX) - tolerance = DIFFICULTY_CONFIG[difficulty].tolerance - isHit = delta <= tolerance MULTI-BOUNDARY WORDS (e.g. "sunflower" = sun|flow|er): - Find the CLOSEST boundary to slashX - Judge against that boundary only RETURN: - null if gesture is null or word not rendered yet - BoundaryHit object (all fields from game.types.ts) This hook is PURE JUDGMENT — no state updates, no animations. Export BoundaryHit result only. Side effects handled elsewhere.
getBoundingClientRect 사용 이유: Vite dev vs production 빌드에서 폰트 렌더링이 미묘하게 다름. CSS transform으로 게임 캔버스를 scale할 경우 좌표 보정도 필요. DOM 기반 계산이 가장 안전.
기대 출력물
  • useSlashJudge.ts — BoundaryHit 반환
  • 멀티 경계 단어 지원 (가장 가까운 경계 자동 선택)
  • 반응형 좌표 실시간 계산
4
단어 렌더러 — WordDisplay 컴포넌트
각 형태소를 개별 span으로 렌더링. 판정 훅이 좌표를 읽을 수 있게.
렌더링
왜 별도 컴포넌트인가: useSlashJudge가 형태소 경계 좌표를 읽으려면 각 형태소가 개별 span 엘리먼트여야 함. 단순 문자열로 렌더링하면 getBoundingClientRect로 경계를 잡을 수 없음.

// Context: POLY M.A.P.S. Morpheme Laser Cutter // useSlashJudge needs individual span refs per morpheme // Create: src/components/WordDisplay.tsx Create WordDisplay component that renders compound words with individually referenceable morpheme spans: PROPS: word: CompoundWord // { id, morphemes: string[], ... } difficulty: Difficulty onBoundaryRef: (index: number, el: HTMLSpanElement | null) => void // called for each boundary position span RENDERING: "raincoat" with morphemes ["rain", "coat"] should render as: rain ← zero-width boundary marker span coat The boundary-N span is the measurement target for useSlashJudge. It should have: - width: 0px, height: 100% - position: absolute or inline-block - data-boundary-index attribute STYLING: - Large font: clamp(48px, 8vw, 80px) - Font weight: 800, letter-spacing: 0.02em - Color: white on dark background - No visible separator between morphemes - Center-aligned in container VISUAL STATE (prop: hitState: 'idle' | 'hit' | 'miss' | 'hint'): - idle: normal white text - hit: each morpheme splits apart with CSS transform translateX - miss: word shakes horizontally (CSS keyframe) - hint: boundary zone glows amber (box-shadow on boundary span) Forward the container ref to parent via forwardRef. Export boundary span positions via onBoundaryRef callback.
zero-width boundary span 전략: 실제 렌더링에 영향 없이 정확한 경계 X좌표를 제공. getBoundingClientRect().left가 곧 boundaryPixelX. 폰트가 바뀌어도 자동 반영.
기대 출력물
  • WordDisplay.tsx — 형태소별 span 구조
  • 4가지 hitState 시각 피드백 (idle/hit/miss/hint)
  • 경계 span ref 콜백으로 외부에 좌표 노출
5
조립 — GameCanvas에서 훅 연결
useSlashInput → useSlashJudge → WordDisplay 전체 파이프라인 통합.
통합
데이터 흐름: 사용자 제스처 → useSlashInput → SlashGesture → useSlashJudge → BoundaryHit → useGameState(score/combo/miss 업데이트) → WordDisplay(hitState 반영)

// Context: POLY M.A.P.S. Morpheme Laser Cutter // All hooks exist: useSlashInput, useSlashJudge, useGameState // WordDisplay component exists // Modify: src/components/GameCanvas.tsx Wire up the complete slash detection pipeline in GameCanvas: PIPELINE ORDER: const canvasRef = useRef(null) const { lastGesture, isSlashing } = useSlashInput(canvasRef) const hit = useSlashJudge(wordRef, currentWord, difficulty, lastGesture) useEffect(() => { if (!hit) return if (hit.isHit) { dispatch({ type: 'HIT', hit }) // triggers score/combo update setHitState('hit') setTimeout(() => nextWord(), 1200) // advance after split animation } else { dispatch({ type: 'MISS', hit }) // increment miss counter setHitState('miss') // Hint progression logic: // missCount === 1: setHitState('hint') after 600ms // missCount === 2: show boundary line guide // missCount >= 3: auto-advance (skip word, no penalty in Easy mode) setTimeout(() => setHitState('idle'), 800) } }, [hit]) LASER TRAIL VISUAL: When isSlashing === true, draw a line from gesture start to current touch position using a overlay (z-index above WordDisplay). - Line color: rgba(251, 191, 36, 0.8) ← amber glow - Line width: 3px with glow effect (shadowBlur: 12, shadowColor: #fbbf24) - Clear canvas on gesture end DIFFICULTY SELECTOR: Show three buttons at bottom: [Easy] [Normal] [Hard] Current difficulty highlighted in amber Changing difficulty resets current game session GameCanvas handles ALL side effects. useSlashInput and useSlashJudge stay pure (no dispatch calls).
Canvas 오버레이 팁: WordDisplay와 별개의 <canvas>를 absolute로 올려야 레이저 궤적이 단어 위에 그려짐. pointer-events: none 필수 — 그렇지 않으면 터치 이벤트를 canvas가 가로챔.
